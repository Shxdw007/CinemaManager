# 🎬 CinemaManager

**CinemaManager** — это многоуровневая автоматизированная система управления кинотеатром, разработанная для полного цикла ведения бизнеса: от настройки залов и составления расписания сеансов до онлайн-продажи билетов конечным зрителям.

### 🕊 Дань уважения
> Данный проект создавался не просто как техническая задача. Это наша дань уважения легендарному кинотеатру **«Звёздный»** в городе Иркутске, который, к сожалению, закрыл свои двери. Спасибо за магию кино. ✨

## 🏗 Архитектура проекта

Проект построен по классической трехзвенной архитектуре (3-Tier) с полным разделением клиентских частей и бизнес-логики:

1. **Backend (Серверное ядро):** `CinemaManager.API`
   * **Стек:** ASP.NET Core (.NET 10), Entity Framework Core, PostgreSQL.
   * **Роль:** Предоставляет RESTful API. Отвечает за безопасность (хэширование паролей BCrypt, JWT-авторизация), маршрутизацию и сложную бизнес-логику (например, валидация временных пересечений сеансов).

2. **Административная панель:** `CinemaManager (WPF)`
   * **Стек:** C#, Windows Presentation Foundation (WPF).
   * **Роль:** Толстый десктопный клиент для персонала кинотеатра. Позволяет управлять контентом без прямого доступа к базе данных (всё общение идет строго по HTTP через паттерн DTO).

3. **Клиентский Web-сайт:** `CinemaManager.Web`
   * **Стек:** Blazor WebAssembly (.NET 10), HTML/CSS.
   * **Роль:** Публичная витрина для зрителей (SPA-приложение). Адаптивный темный дизайн, просмотр афиши и интерактивная сетка выбора мест.

## 🚀 Основной функционал

### 🛡 Для администраторов (WPF Client)
* **CRUD операции:** Полное управление фильмофондом (название, постеры, длительность) и конфигурацией физических залов (ряды, места, типы залов).
* **Умное расписание:** Создание сеансов с автоматическим расчетом времени окончания (длительность фильма + 20 минут на уборку зала).
* **Защита от овербукинга:** Серверная валидация, блокирующая создание пересекающихся по времени сеансов в одном зале.
* **Статистика (в разработке):** Аналитика продаж и заполняемости залов.

### 🍿 Для зрителей (Blazor SPA)
* **Афиша и расписание:** Просмотр списка актуальных фильмов и доступных сеансов (открытый доступ без авторизации).
* **Интерактивное бронирование:** Визуальный выбор посадочных мест на сгенерированной схеме зала.
* **Личный кабинет:** Форма авторизации и просмотр истории покупок (электронных билетов с QR-кодами).

## 🛠 Запуск проекта (Для разработчиков)

1. Убедитесь, что у вас установлен **.NET 10 SDK** и запущен сервер **PostgreSQL**.
2. В файле `CinemaManager.API/appsettings.json` пропишите актуальные данные для подключения к вашей БД (`DefaultConnection`).
3. Примените миграции для создания таблиц БД (через Package Manager Console):
   ```powershell
   Update-Database
4. Запустите проект CinemaManager.API. В открывшемся Swagger выполните POST /api/users/seed-admin для создания дефолтного аккаунта администратора.
5. Настройте мульти-запуск (Multiple Startup Projects) в Visual Studio для проектов CinemaManager.API и CinemaManager.Web.
6. Десктопный WPF-клиент запускается отдельно для входа под учетной записью администратора.
